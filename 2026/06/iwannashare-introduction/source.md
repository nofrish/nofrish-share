我做 IWannaShare，是因为我越来越频繁地遇到同一个问题：和 AI 聊完以后，真正有价值的部分往往不是一句话，而是一段结构完整的结论、判断、清单或笔记。

这些内容适合被保存，也适合被分享。但直接复制一大段聊天记录，通常并不好看；截图又太随意，长内容还会断裂。于是我想要一个更自然的出口：把 AI 生成的文字，变成一张漂亮、清晰、适合手机阅读的长图，或者变成一个可以公开访问的静态网页。

> [!KEY]
> IWannaShare 的目标不是做一个复杂的排版软件，而是给 AI Agent 一个很顺手的分享工具：输入 Markdown，输出可以直接分享的图片或网页。

## 它解决什么问题

很多 AI 对话最后都会沉淀成几类内容：

- 一个经过讨论后的判断
- 一份推荐理由和取舍说明
- 一段可以转发给朋友的解释
- 一篇读书、观影、旅行或产品笔记
- 一个临时但值得留下来的想法

这些内容如果只是留在聊天窗口里，很快就会消失在上下文中。如果要发给别人，又需要重新整理格式、控制长度、处理标题和重点。

IWannaShare 想做的事情很简单：让这一步变成一个命令。

## 它现在能做什么

当前版本主要支持两种输出。

第一种是长图。它会把 Markdown 渲染成一张 1080px 宽的 PNG 图片，适合发到微信、群聊、朋友圈、笔记软件或任何手机阅读场景。图片会使用统一的阅读主题，有标题、正文、列表、引用、代码和 callout 的样式。

第二种是网页。它可以把同一份 Markdown 发布成一个静态网页，并通过 GitHub Pages 得到一个公开链接。这样别人不需要保存图片，只要打开链接，就能阅读完整内容。

> [!NOTE]
> 这两个输出不是两个完全不同的系统。它们共享同一种输入方式：Markdown。对 Agent 来说，只需要先把最终内容整理成 Markdown，再交给 IWannaShare。

## 为什么是 Markdown

Markdown 足够简单，也足够通用。AI 很擅长生成 Markdown，人也很容易检查和修改。

我希望这个工具的输入不要变成一个复杂表单，也不要要求用户理解太多前端概念。只要内容本身是清楚的，Agent 就可以把它整理成适合发布的结构。

标准 Markdown 已经能覆盖大部分场景：标题、段落、列表、加粗、代码、引用和表格。除此之外，IWannaShare 还支持少量 Markdown+ 写法，比如 callout：

```md
> [!KEY]
> 这里写核心结论。

> [!NOTE]
> 这里写补充说明。
```

这些标记不会原样出现在最终内容里，而是会被渲染成不同的视觉块。也就是说，读者看到的是样式，而不是 `KEY` 或 `NOTE` 这样的标签。

## 它更像一个 Agent 工具

IWannaShare 不是从“人打开一个软件开始排版”这个角度设计的，而是从 Agent 的工作流出发设计的。

一个典型流程是这样的：

1. 用户和 AI 讨论一个问题。
2. AI 把最终结论整理成 Markdown。
3. AI 调用 `iws` 命令生成长图或发布网页。
4. 用户直接粘贴图片，或者把链接发给别人。

这意味着它不只是一个 CLI，也是一种分享协议。它把“内容生成”和“内容交付”之间的最后一步补上了。

## 我在意的设计方向

IWannaShare 目前只有一个默认主题。这是有意的。

我不想一开始就做很多主题选择，因为太多选择会让工具变得松散。现在更重要的是先把一个主题打磨到足够稳定：

- 手机上读起来不费力
- 左右留白不过度浪费空间
- 标题有设计感，但不过分抢内容
- 正文有足够的重量和行距
- callout 能突出重点，但不刺眼
- 整体像一篇可以被认真阅读的短文，而不是一张花哨海报

未来当然可以扩展更多主题，但第一版应该先把“默认体验”做好。

## 目前的使用方式

生成图片时，可以直接把 Markdown 通过 stdin 传给工具：

```bash
iws --copy --open <<'MARKDOWN'
---
title: 一段值得分享的内容
---

这里写 Markdown 内容。
MARKDOWN
```

如果要发布成网页，可以使用：

```bash
iws publish --copy-url <<'MARKDOWN'
---
title: 一段公开分享的内容
slug: optional-slug
---

这里写 Markdown 内容。
MARKDOWN
```

`--copy` 会把图片复制到 macOS 剪切板，用户可以直接粘贴。`--open` 会在生成后打开结果，方便马上检查效果。`publish` 则会把内容写入配置好的 GitHub Pages 仓库，提交并推送，然后得到一个公开链接。

## 我希望它变成什么

我希望 IWannaShare 最后变成一个很轻的基础设施：当 AI 产出一段值得分享的内容时，用户不需要再思考怎么排版、怎么截图、怎么保存、怎么发出去。

它应该安静地完成最后一步，把内容变成一个适合传播的形态。

> [!KEY]
> 对我来说，IWannaShare 的价值不是“生成一张图”，而是让 AI 对话里的好内容有一个自然、漂亮、低摩擦的出口。
